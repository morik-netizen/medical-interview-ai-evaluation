# 整骨院実習 運用フロー・スマホUI設計書

## 概要

本書は、整骨院臨床実習においてPlaudボイスレコーダー1台を4〜6人の学生で共有し、
GAS WebApp経由でAI医療面接評価を行うための、実践的な運用フロー・UI設計をまとめたものである。

---

## 1. 整骨院での1日の運用フロー

### 1-1. 1日のタイムライン（半日・午前の例）

```
08:30  学生集合・着替え・朝礼
08:40  【初日のみ】システム説明・QRコード読み取り・ブックマーク登録（5分）
08:45  実習開始（施術見学・補助業務）
       ┃
       ┃  ※患者の面接機会が来た学生から順番にPlaudを使用
       ┃
       ┃  学生A: 09:00-09:10 面接 → 09:10-09:20 文字起こし待ち＋AI評価
       ┃  学生B: 09:10-09:20 面接 → 09:20-09:30 文字起こし待ち＋AI評価
       ┃  学生C: 09:30-09:40 面接 → 09:40-09:50 文字起こし待ち＋AI評価
       ┃  学生D: 09:50-10:00 面接 → 10:00-10:10 文字起こし待ち＋AI評価
       ┃  （面接の合間は全員通常の実習業務を継続）
       ┃
11:30  実習業務終了
11:30  結果の確認・振り返りタイム（まだ見ていない学生はここで確認）
12:00  終了・片付け
```

### 1-2. Plaud1台の回し方（ローテーション方式）

```
【基本ルール】
- Plaudは「面接担当テーブル」の決まった位置に常設
- 面接する学生がPlaudを操作（録音開始/停止）
- 面接終了後、Plaudはそのまま設置場所に戻す
- 文字起こしは面接後に各自のスマホで行う（後述）

【順番の決め方】
- 朝礼時に「今日の面接順」を決める（患者の来院タイミング次第で柔軟に）
- 指導者が「次は○○さん、面接お願い」と指示
- 面接機会がない時間帯は通常の実習業務（施術補助・見学・カルテ整理等）
```

### 1-3. 実習の他業務との両立

```
【面接していない時間の過ごし方】
- 通常の実習業務を継続（施術補助、見学、消毒・片付け等）
- AI評価の結果確認は「手が空いた隙間時間」または「振り返りタイム」で
- 面接テキストの匿名化作業（3〜5分）も隙間時間でOK
- 他の学生の面接中にスマホを触らない（患者さんの前では失礼）

【指導者への配慮】
- 「AI評価を見る時間」は指導者に事前に説明しておく
- 「振り返りタイム」を実習スケジュールに公式に組み込むのがベスト
- スマホ操作は待合室や休憩室など、患者から見えない場所で
```

## 2. Plaud1台運用の具体的な段取り

### 2-1. 事前準備（実習初日に1回だけ）

| 項目 | 担当 | 内容 |
|------|------|------|
| Plaudのアカウント | 教員 | 教員のアカウントでPlaudをセットアップ。学生個人のアカウントは使わない |
| Plaudアプリ | 教員のスマホ | 教員のスマホ1台にPlaudアプリをインストール・ログイン |
| Plaud本体のBluetooth接続 | 教員 | 教員のスマホとPlaudをBluetooth接続しておく |
| 設置場所の確認 | 教員＋指導者 | 面接テーブルのどこにPlaudを置くか決定（患者と学生の間、テーブルの端等） |
| QRコード配布 | 教員 | GAS WebAppのURLをQRコード化し、印刷して整骨院に掲示 |

### 2-2. アカウント管理方針

```
【推奨：教員のアカウント1つで運用】

理由：
1. 学生ごとにアカウントを作ると切り替えが煩雑
2. Plaudのクラウドに全録音データが残るので、教員が管理すべき
3. 文字起こしテキストの共有は「テキストのコピペ」で十分

運用フロー：
- Plaudで録音 → 教員のスマホに自動同期 → 文字起こし実行
- 文字起こし完了後、教員が該当テキストを学生に共有
  （LINEグループ、AirDrop、テキストファイル共有等）
```

### 2-3. 面接前の準備

```
所要時間：約1分

1. 教員（または前の学生）がPlaudの設置位置を確認
   - テーブルの患者寄り中央（患者・学生の声が均等に入る位置）
   - Plaud NotePin の場合：テーブルに平置き（クリップは使わない）

2. 前の録音が完了していることを確認
   - Plaud本体のLEDで録音中でないことを確認

3. 患者への説明（学生が行う）
   「学習のために面接の音声を録音させていただいてもよろしいでしょうか。
    録音は学習目的のみに使用し、個人情報は匿名化して管理いたします。」
   ※ 患者が拒否した場合は録音しない。面接自体は実施する
```

### 2-4. 面接中の操作

```
所要時間：面接5〜10分

【録音開始】担当学生が操作
- Plaud本体のボタンを1回押す（LEDが点灯＝録音開始）
- 面接を開始

【面接中】
- Plaudには触れない
- 通常通り面接を実施

【録音停止】担当学生が操作
- 面接終了後、Plaud本体のボタンを1回押す（LED消灯＝録音停止）
```

### 2-5. 面接後の文字起こしとテキスト転送

```
所要時間：約5〜8分（文字起こし待ち含む）

【Step 1】録音データの同期（教員のスマホ）
- Plaudアプリを開く → Bluetooth接続で自動同期（1〜2分）
- 同期完了後、該当の録音を選択

【Step 2】文字起こし実行（教員のスマホ）
- Plaudアプリで「文字起こし」をタップ
- 言語：日本語
- 話者分離：ON
- 処理時間：約2〜5分（面接の長さによる）

【Step 3】テキストの取り出し（教員のスマホ → 学生のスマホ）

方法A（推奨）：テキスト共有
- Plaudアプリで文字起こし結果を表示
- 「共有」ボタン → テキストをコピー
- LINEで学生に送信（個別メッセージ）

方法B：教員がその場で読み上げ・スクショ
- 学生が教員のスマホ画面を見てスクリーンショット
- スクショからテキストをコピー

方法C：メール送信
- Plaudアプリからメールで送信 → 学生がメールで受信

【Step 4】次の学生への引き継ぎ
- 「文字起こし完了したよ、次の人どうぞ」と声かけ
- Plaudは設置場所に戻す（動かさない）
```

### 2-6. 待ち時間の活用

```
文字起こし待ち（2〜5分）の間に学生ができること：
- テキストの匿名化作業の準備（どの部分を消すか考える）
- 面接の自己振り返り（記憶が新しいうちに）
- 通常の実習業務に戻る
```

## 3. 学生の1回利用フロー（ステップバイステップ）

### 全体所要時間：約20〜25分

```
 面接         文字起こし待ち    匿名化      AI評価     結果確認
┣━━━━━━━┫┣━━━━━━━━┫┣━━━━━┫┣━━━━━┫┣━━━━━━┫
 5-10分        3-5分           3-5分       1-2分      5-10分
              (この間は通常業務)
```

### Step 1：面接の実施（5〜10分）

```
1. 指導者から面接の指示を受ける
2. Plaudの録音ボタンを押す（1秒）
3. 患者との面接を実施（5〜10分）
4. 面接終了後、Plaudの録音ボタンを押して停止（1秒）
5. 患者にお礼を言い、通常業務に戻る

つまずきポイント：
  × 録音ボタンの押し忘れ
  → 対策：面接前に「録音開始チェック」を習慣化。LEDの点灯を確認

  × 録音停止忘れ（次の人の面接まで録音が続く）
  → 対策：面接後すぐに停止。教員も声かけでリマインド
```

### Step 2：文字起こしテキストの受け取り（3〜5分）

```
1. 教員に「面接終わりました」と報告
2. 教員がPlaudアプリで文字起こしを実行
3. 完了後、LINEでテキストが送られてくる
4. テキストを受信したら確認

（この間は通常の実習業務を継続）

つまずきポイント：
  × 文字起こしに時間がかかり焦る
  → 対策：「5分くらいかかるから、その間は別の作業をしていてね」と事前説明

  × テキストが文字化けしている
  → 対策：Plaudアプリの言語設定を日本語に確認
```

### Step 3：テキストの匿名化（3〜5分）

```
1. 受け取ったテキストをスマホのメモアプリにコピー
2. 以下を手動で置き換え：
   - 患者の氏名 → 「患者A」
   - 学生の氏名 → 「術者」
   - 住所・電話番号等 → 削除
   - 整骨院名 → 削除（言及されていれば）
3. 匿名化したテキストをコピー

つまずきポイント：
  × どこまで匿名化すべきかわからない
  → 対策：匿名化チェックリストをシステム画面内に表示（後述UI設計参照）

  × スマホでのテキスト編集が面倒
  → 対策：最低限「氏名」だけ置換すればOKと伝える。完璧を求めない
```

### Step 4：AI評価の実行（1〜2分）

```
1. スマホブラウザでシステムにアクセス（ブックマーク or ホーム画面）
2. 「匿名化済みテキスト」欄に貼り付け
3. レベルを確認（システム推奨＋自分で変更可）
4. 「AI評価を実行」ボタンをタップ
5. ローディング画面を見ながら30秒〜1分待つ

つまずきポイント：
  × テキストの貼り付けがうまくいかない
  → 対策：テキストエリアをタップ → 「ペースト」の手順を初回に説明

  × 通信エラーで評価が失敗
  → 対策：WiFi接続を確認。ダメなら携帯回線に切り替え。再度実行
```

### Step 5：結果の確認（5〜10分）

```
1. 評価結果が表示される
2. まず「サマリーカード」で全体像を把握（10秒）
3. 「最優先アクション」を確認（30秒）
4. 気になるカテゴリの詳細を折りたたみで展開（2〜5分）
5. 「次回の目標」をスクリーンショットで保存

つまずきポイント：
  × 情報量が多くて何を見ればいいかわからない
  → 対策：Level 1は項目数を絞り褒め中心。Level 2でも最優先アクション1つに集中

  × 評価が低くて落ち込む
  → 対策：レベル設計で「成長中」表記を採用。改善点は1つずつ提示
```

#### テキスト入力のスマホ最適化

```css
/* スマホでのズーム防止（iOS Safari対策） */
textarea {
  font-size: 16px;        /* 16px未満だとiOSが自動ズームする */
  -webkit-text-size-adjust: 100%;
}

/* テキストエリアの使いやすさ向上 */
textarea {
  width: 100%;
  min-height: 200px;
  padding: 16px;
  border: 2px solid #d1d5db;
  border-radius: 12px;
  font-family: sans-serif;  /* 等幅フォントはスマホで読みにくい */
  line-height: 1.8;
  resize: none;             /* スマホではリサイズ不要 */
}

/* フォーカス時に目立つ枠 */
textarea:focus {
  border-color: #2563eb;
  box-shadow: 0 0 0 4px rgba(37, 99, 235, 0.15);
  outline: none;
}
```

#### コピペの最適な方法

```
【学生への案内】

1. LINEで受信したテキスト（教員から送られたもの）を長押し
2. 「コピー」をタップ
3. 本システムのテキスト欄をタップ
4. テキスト欄を長押し → 「ペースト」をタップ

※ メモアプリで匿名化してから貼り付ける場合：
  メモアプリでテキスト全選択 → コピー → 本システムに切替 → ペースト
```

### 4-5. レベル推奨ロジック

```javascript
function getRecommendedLevel(history) {
  const count = history.length;

  // 初回〜2回目 → Level 1
  if (count <= 2) return 1;

  // Level 1で7/10以上を2回達成 → Level 2を推奨
  const level1Results = history.filter(h => h.level === 1);
  const level1HighCount = level1Results.filter(h => h.checkCount >= 7).length;
  if (level1HighCount >= 2) {
    // Level 2で80点以上を3回達成 → Level 3を推奨
    const level2Results = history.filter(h => h.level === 2);
    const level2HighCount = level2Results.filter(h => h.score >= 80).length;
    if (level2HighCount >= 3) return 3;
    return 2;
  }

  return 1;
}
```
