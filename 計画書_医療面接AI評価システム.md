# 医療面接AI評価システム 計画書

## 1. プロジェクト概要

### 目的
柔道整復学科の学生が、実際の患者との医療面接やロールプレイの音声を録音し、
AIが自動で評価・改善点を提示する**セルフチェックシステム**を構築する。

### コンセプト
```
学生がPlaudで医療面接を録音
        ↓
  音声データをシステムにアップロード
        ↓
  自動文字起こし＋話者分離
        ↓
  AIが評価表に基づき自動判定
        ↓
  評価レポート出力
  ・各項目のGood/Poor判定
  ・総合フィードバックコメント
  ・該当技法と紐づけた具体的アドバイス
  ・スコアリング（点数化）
```

### 2-2. 技術スタック案

| レイヤー | 技術候補 | 理由 |
|---|---|---|
| フロントエンド | Next.js（React） | SSR対応、モダンUI |
| バックエンド | Python（FastAPI） | AI/ML系ライブラリ豊富 |
| 文字起こし | Plaud API → 話者分離済みテキスト取得 | Plaudの精度を活用 |
| 文字起こし（代替） | OpenAI Whisper API | Plaud APIが使えない場合 |
| 話者分離（代替） | pyannote.audio | Whisper使用時の補完 |
| AI評価エンジン | Claude API（Sonnet 4.6） | 日本語の理解力、長文処理 |
| データベース | PostgreSQL or Supabase | 履歴管理 |
| ファイルストレージ | S3互換（Cloudflare R2等） | 音声ファイル保存 |
| ホスティング | Vercel + Railway or Render | 低コスト運用 |

## 3. 評価設計（最重要）

### 3-1. 評価の段階的アプローチ

学生の習熟度に応じて段階的に評価レベルを上げる設計とする。

| レベル | 対象 | 評価項目数 | 用途 |
|---|---|---|---|
| Level 1（入門） | 初学者 | 10項目 | Step1（導入）の練習 |
| Level 2（基本） | Normal版ベース | 30項目 | 面接全体の基本評価 |
| Level 3（応用） | Hard版ベース | 68項目 | 本格的な評価・OSCE対策 |

### 3-2. 評価項目の分類

音声テキストから評価できるものと、できないものを明確に分離する。

#### A. テキストから自動評価できる項目（主要評価対象）

| カテゴリ | 評価項目 | 判定方法 |
|---|---|---|
| **面接の流れ** | Step1→2→3→4の順序性 | 対話の構造分析 |
| **オープニング** | 挨拶の有無・適切さ | キーワード検出＋文脈 |
| | 自己紹介の有無 | キーワード検出 |
| | 患者確認（フルネーム） | パターン検出 |
| | 面接の目的説明と同意取得 | 文脈分析 |
| **質問技法** | 開かれた質問の使用頻度と適切さ | 質問文の分類 |
| | 閉ざされた質問の使用頻度と適切さ | 質問文の分類 |
| | 質問の種類バランス（OQ/CQ比率） | 統計分析 |
| | 焦点を絞った質問の活用 | 文脈分析 |
| **傾聴技法** | 促し（「なるほど」「それで」等） | キーワード＋頻度 |
| | 繰り返し（reflection） | 患者発言との照合 |
| | 要約（summarization） | 要約パターン検出 |
| | 確認（confirmation） | 確認パターン検出 |
| | バックトラッキング | パターン検出 |
| **共感・支持** | 共感の言葉がけ | キーワード＋文脈 |
| | 支持・勇気づけの言葉 | キーワード＋文脈 |
| **情報聴取** | 主訴の把握 | 情報項目チェック |
| | 発生機序 | 質問有無 |
| | 部位・性状・程度・経過 | 質問有無 |
| | 増悪・寛解因子 | 質問有無 |
| | 随伴症状 | 質問有無 |
| | ADLへの影響 | 質問有無 |
| | 既往歴・家族歴・社会歴 | 質問有無 |
| | 解釈モデルの聴取 | LET'S HEARの要素検出 |
| **言葉遣い** | 敬語の適切さ | 文法チェック |
| | 専門用語の多用チェック | 専門用語辞書照合 |
| | 患者の話を遮っていないか | 発話の重なり分析 |
| **締めくくり** | 最終要約と確認 | パターン検出 |
| | 聞き漏らし確認 | キーワード検出 |
| | 締めの挨拶 | キーワード検出 |

#### B. 音声分析で補完評価できる項目

| カテゴリ | 評価項目 | 判定方法 |
|---|---|---|
| **声の調子** | 声の大きさ・安定性 | 音量レベル分析 |
| | 話すスピード | 発話速度計測 |
| | 抑揚の有無 | ピッチ変動分析 |
| | 沈黙の活用 | 無音区間の検出・長さ |
| **対話バランス** | 術者と患者の発話時間比率 | 話者別の時間計測 |
| | 患者を遮るタイミング | 発話重複検出 |

#### C. テキスト・音声では評価困難な項目（自己申告 or 対象外）

| カテゴリ | 評価項目 | 対応方法 |
|---|---|---|
| 身だしなみ | 清潔感・服装等 | セルフチェックリスト（別途） |
| 視線・表情 | アイコンタクト等 | 将来的にカメラ連携で対応可能 |
| 姿勢・態度 | 身の置きかた等 | 同上 |
| 座席配置 | 90度法等 | セルフチェック項目として |

## 4. AI評価エンジンの設計

### 4-1. プロンプト設計の基本方針

AIへの入力は以下の3層で構成する。

```
【System Prompt（固定）】
  ・あなたは柔道整復学科の医療面接指導教員です
  ・評価基準（テキストの内容をすべて組み込み）
  ・評価表の全項目定義
  ・スコアリングルール
  ・出力フォーマット指定

【Knowledge Base（固定）】
  ・テキスト全文（医療面接テキスト整理.md）
  ・38種の技法の定義と具体例
  ・LET'S HEARフレームワーク
  ・質問の分類基準
  ・会話の良い例・悪い例

【User Input（毎回変動）】
  ・話者分離済みの面接テキスト
  ・面接メタ情報（日時、種別、学生レベル）
  ・音声分析結果（発話時間、沈黙、速度等）
  ・前回の評価結果（差分比較用）
```

### 4-2. 評価の信頼性を高める工夫

1. **構造化出力**: AIにJSON形式で出力させ、パースして表示
2. **根拠の明示**: 各判定に対して該当する発話箇所を引用
3. **二段階評価**:
   - 1回目: 項目ごとの判定（Good/Poor＋根拠）
   - 2回目: 総合コメント＋改善アドバイス生成
4. **評価のキャリブレーション**: 模範面接データで事前テストし精度調整

## 7. データプライバシーと倫理的配慮

### 7-1. 患者のプライバシー保護
- 録音前に**患者の同意**を必ず取得（書面）
- 音声データは匿名化処理（患者名を自動マスキング）
- データはサーバーに保存後、一定期間で自動削除
- HTTPS通信、データベース暗号化

### 7-2. 学生のプライバシー
- 評価結果は本人のみ閲覧可能
- 教員への共有は学生の同意制

### 7-3. AI評価の位置づけ
- AIの評価はあくまで**参考・自己学習支援**であり、成績評価には直接使用しない
- 最終的な評価は教員が行う
- AI判定に対する異議申し立ての仕組みを設ける
